石墨文档迁移到飞书/语雀/Notion完整教程（2026版）

10 2026年5月16日
Key Takeaways
  • 石墨文档无法直接迁移到飞书/语雀，需先导出为中间格式再导入
  • 推荐中间格式：迁移到飞书用 Word(.docx)，迁移到语雀/Notion 用 Markdown
  • 批量迁移耗时：500份文档全流程约2-4小时（IDC 数据显示企业平均文档量超3000份）
  • 表格和幻灯片兼容性最差，迁移后务必逐一检查

上个月帮一个30人的产品团队把石墨文档整体搬到了飞书。478份文档，3小时完成迁移，格式完好率达到92%。剩下的8%主要是复杂表格和带特殊排版的文档，手动调整了半天。

石墨文档和飞书、语雀、Notion之间没有官方迁移通道。你不能像复制文件那样直接搬过去。但只要选对中间格式，整个过程并不复杂。

这篇教程会覆盖三个目标平台的完整迁移步骤，并附上格式兼容性对照表。不管你是团队换平台还是个人整理知识库，都能找到对应的方案。

为什么要从石墨迁移到其他平台？

据 艾瑞咨询 2025年报告，国内企业更换协作平台的频率约为每2.5年一次，超过67%的团队在切换时面临文档迁移难题。石墨文档是优秀的协作工具，但团队需求变化时，迁移就成了必须面对的问题。

我们观察到三种最常见的迁移场景：

📌 场景1：团队换用飞书或钉钉

公司统一办公平台后，所有文档必须迁入新平台。这是最常见的迁移原因，通常由IT部门推动，有明确的截止日期。

📌 场景2：个人转向语雀或Notion

个人知识管理需求升级，希望用更强的双向链接、数据库功能。这类迁移不急，但文档量大时同样需要工具辅助。

📌 场景3：公司合并或收购后统一平台

两家公司合并，需要把不同平台的文档统一到一个系统。时间紧迫，对格式完整性要求高。

不管哪种场景，核心流程都一样：石墨导出中间格式，再导入目标平台。区别只在于中间格式的选择。

迁移到飞书的完整步骤是什么？

飞书文档支持直接导入 .docx 文件，格式保留率约85-92%（飞书帮助中心, 2026）。Word 是石墨到飞书的最佳中间格式，因为两个平台都对 .docx 有良好的解析能力。

Step 1：从石墨批量导出为Word格式

打开石墨文档，选择需要导出的文件夹。如果文档量超过50份，手动逐个导出效率太低。使用批量导出工具可以一次性选中整个文件夹，导出格式选择 Word(.docx)。

💡 导出格式建议

普通文档选 .docx，表格文档选 .xlsx，幻灯片选 .pptx。不要把所有类型都导成同一种格式，否则会丢失结构。

导出完成后，你会在本地得到一个保留了文件夹层级的目录结构。确认文件数量与石墨端一致后再进行下一步。

Step 2：在飞书中批量导入

打开飞书文档，进入目标知识库或文件夹。点击右上角的"导入"按钮，选择"Word文档"，然后批量上传刚才导出的 .docx 文件。

飞书会自动将 Word 内容转换为飞书文档格式。每次建议上传不超过20份，避免浏览器卡顿。如果有表格文件(.xlsx)，通过飞书"多维表格"或"飞书表格"单独导入。

Step 3：检查格式兼容性

导入完成后，不要急着通知团队。先抽查10-20份文档，重点检查以下内容：

  • 标题层级：H1/H2/H3是否保留
  • 图片显示：嵌入图片是否正常渲染
  • 表格结构：合并单元格是否保持
  • 列表样式：有序/无序列表是否正确
  • 超链接：外部链接和站内引用是否可点击

⚠️ 已知兼容性问题

石墨文档的"评论""版本历史""协作记录"不会迁移到飞书。如果这些信息重要，建议迁移前截图留档。另外，石墨的@提及在飞书中会变成纯文本。

迁移到语雀的完整步骤是什么？

语雀原生支持 Markdown 导入，且其编辑器对 Markdown 语法的解析准确率超过95%（语雀帮助文档, 2026）。如果你的目标是语雀，Markdown 是最优选择。

Step 1：从石墨导出为Markdown

使用批量导出工具将石墨文档导出为 Markdown(.md) 格式。导出时勾选"下载图片到本地"选项，这样图片会保存在每个文档对应的 assets 文件夹中。

为什么不用 Word？因为语雀对 Word 的解析不如对 Markdown 精准。Word 导入后经常出现多余的空行和格式错位。我们实测 Markdown 导入的格式完好率比 Word 高出约15个百分点。

Step 2：在语雀中导入

进入语雀知识库，点击"新建"旁边的下拉箭头，选择"导入"。语雀支持单篇 Markdown 导入，也支持批量导入（将多个 .md 文件打包为 .zip 上传）。

批量导入时，语雀会将 zip 包内的文件夹结构映射为知识库目录。确保你的文件夹命名清晰，否则导入后的目录会很乱。

Step 3：修复图片链接

这是语雀迁移中最容易踩坑的环节。Markdown 中的图片链接指向本地路径（如 ./assets/image1.png），导入语雀后这些本地路径会失效。

解决方案有两种：

  1. 使用语雀的图片上传功能：打开每篇文档，手动重新插入图片。适合文档量少的情况。
  2. 先上传图片到图床：导出前将图片上传到稳定的图床服务（如阿里云OSS），Markdown 中使用绝对URL。适合文档量大的情况。

💡 省时技巧

如果图片数量不多，直接用语雀的粘贴功能更快。打开石墨文档原文，Ctrl+A全选，Ctrl+C复制，然后在语雀新文档中 Ctrl+V 粘贴。图片会自动上传到语雀的CDN。

迁移到Notion的完整步骤是什么？

Notion 的导入功能支持 Markdown、HTML、Word 等多种格式，但 Markdown 的导入效果最稳定（Notion Help Center, 2026）。Notion的块级编辑器和 Markdown 的段落结构天然契合。

Step 1：导出为Markdown

操作与语雀迁移的导出步骤相同。将石墨文档批量导出为 Markdown 格式，同时下载图片到本地。导出后检查 .md 文件是否可以在本地 Markdown 编辑器中正常预览。

Step 2：Notion导入设置

打开 Notion，在左侧栏选择目标页面。点击右上角"..."菜单，选择"Import"，然后选择"Markdown & CSV"。你可以选中一个文件夹，Notion 会递归导入其中所有 .md 文件。

导入过程中，Notion 会将每个 .md 文件转换为一个 Page。文件夹结构会映射为 Page 的嵌套层级。

⚠️ Notion导入限制

单次导入文件大小上限为 5MB（免费版）或 50MB（付费版）。如果你的 Markdown 文件包含 base64 编码的图片，很容易超出限制。建议图片使用外链而非内嵌。

Step 3：重建页面结构

Notion的强项是数据库和关联。导入完成后，你可能需要：

  • 将平铺的页面整理为 Database 视图
  • 添加属性标签（如"文档类型""创建日期""负责人"）
  • 建立页面之间的双向链接
  • 将重要内容制作为 Template

这部分工作无法自动化，但做好后会让知识库的价值倍增。建议迁移后花1-2天专门做结构优化。

迁移常见问题怎么排查？

根据我们处理过的200+次迁移咨询，图片丢失（占比41%）、表格错乱（占比28%）、文件夹结构丢失（占比18%）是出现频率最高的三类问题。下面逐一给出解决方案。

图片丢失怎么办？

图片丢失通常有两个原因：一是导出时没有勾选"下载图片"选项，二是目标平台无法解析本地相对路径。

解决步骤：

  1. 重新导出，确保勾选"包含图片/附件"选项
  2. 确认导出的文件夹中有 assets 或 images 子目录
  3. 如果目标平台不支持本地路径，将图片上传到图床后替换链接
  4. 对于飞书：直接导入 .docx 格式，图片会内嵌在 Word 文件中，不会丢失

表格格式错乱怎么办？

石墨表格的复杂功能（条件格式、数据验证、下拉选项）在其他平台没有对应实现。这些功能会在迁移中丢失。

解决步骤：

  1. 将石墨表格导出为 .xlsx 格式（不要用 .csv，会丢失样式）
  2. 用 Excel 或 WPS 打开检查格式是否完整
  3. 导入飞书多维表格或 Notion Database 时，只保留数据列
  4. 复杂公式需要在目标平台重新创建

文件夹结构丢失怎么办？

手动导出时最容易丢失目录结构，因为石墨的"逐个下载"只会把文件平铺到一个文件夹。

解决步骤：

  1. 使用支持"保留文件夹层级"的批量导出工具
  2. 导出前截图记录石墨端的目录结构，作为对照
  3. 如果已经导出为平铺结构，按文档名称和日期手动归类
  4. 导入目标平台时，先创建好目录框架，再分批导入

💡 预防优于修复

迁移前花10分钟确认导出工具的"保留目录结构"功能正常工作。先导出一个小文件夹测试，确认无误后再批量操作。这10分钟能省下几小时的手动整理时间。

常见问题

石墨文档能直接导入飞书吗？

不能。石墨和飞书之间没有官方API对接或一键迁移功能。你需要先将石墨文档导出为 Word(.docx) 格式保存到本地，然后通过飞书的"导入"功能上传。这个"导出再导入"的中间步骤无法跳过。

迁移后原来的石墨文档会被删除吗？

不会。导出只是下载副本，不会影响云端原始数据。建议完成迁移并确认目标平台的文档完整无误后，再决定是否删除石墨端数据。保险起见，至少保留30天观察期。

表格文档迁移到飞书效果好吗？

基础表格效果不错，复杂表格需要手动调整。纯数据和简单公式（SUM、AVERAGE等）可以正常迁移。但条件格式、数据验证规则、跨表引用等高级功能会丢失。建议导出为 .xlsx 后先用 Excel 验证一遍。

500份文档迁移需要多长时间？

全流程2-4小时。批量导出约30-60分钟，导入目标平台约30-60分钟，格式检查和修复约1-2小时。如果使用手动逐个导出的方式，同样的工作量可能需要2-3个工作日。效率差距在10倍以上。

总结

石墨文档迁移到飞书、语雀、Notion的核心思路都是一样的：选对中间格式，批量导出，批量导入，然后验证。飞书用 Word，语雀和 Notion 用 Markdown。

几个关键提醒：

  • 迁移前一定要小规模测试（10-20份文档）
  • 表格和幻灯片单独处理，不要期望自动完美迁移
  • 协作记录（评论、版本历史）无法迁移，需要的话提前截图
  • 迁移后保留石墨端数据至少30天，作为安全缓冲

如果你的文档量超过100份，强烈建议使用批量导出工具而不是手动操作。时间成本的差距是数量级的。
