    石墨文档迁移到飞书文档完整教程：官方导入入口、文件夹迁移与格式检查

    14 2026年7月15日
    团队迁移关键结论
    • 飞书有官方石墨导入相关说明，但迁移前仍要先把石墨文档下载到本地
    • 普通文档迁移到飞书优先 Word，表格优先 Excel，知识库文本可用 Markdown
    • 团队迁移要先确认权限、范围、失败文件和验收人，不要直接全量开跑
    • 评论、历史版本、协作记录和@提及通常无法完整迁移
    • 大批量迁移建议分批导入，并保留至少 30 天石墨端回滚窗口

    石墨文档迁移到飞书文档的稳妥做法是：先确认石墨空间、文件夹、成员权限和导出限制；再把普通文档导出为 Word 或 Markdown、表格导出为 Excel、演示文稿导出为 PPTX/PDF；然后在飞书云文档中选择导入为在线文档或上传本地文件夹；最后检查标题、图片、表格、链接和权限。飞书官方支持导入多种本地文件类型，但团队迁移仍需要分批、验收和回滚计划。

    这篇文章专门面向“公司要统一到飞书”的迁移场景。和旧的跨平台总览不同，本文会围绕飞书官方入口、文件夹上传、开放平台导入任务和团队验收展开。

    官方资料依据

    飞书帮助中心提供“将石墨文档导入为云文档”和“上传或导入本地文件和文件夹”的说明；飞书开放平台也提供创建导入任务接口，可把 Word、TXT、Markdown、Excel 等文件导入为飞书在线云文档。参考：将石墨文档导入为云文档、上传或导入本地文件和文件夹、导入文件概述。

    飞书是否支持导入石墨文档？

    支持，但不要理解成“输入石墨账号后一键搬家”。飞书官方文档的前提是，你已经把石墨在线文档下载到本地，并选择飞书可导入的文件类型。也就是说，迁移链路仍然是：石墨导出本地文件，再导入飞书云文档。

    官方石墨导入入口适合什么场景

    适合已经整理好本地文件的团队，尤其是 Word、Excel、Markdown、PPTX 等格式明确的资料。对于大批量迁移，建议先用小文件夹测试导入效果，再按部门或项目批量处理。

    为什么仍需要先做好石墨批量导出

    飞书导入解决的是“把本地文件转成飞书云文档”，它不负责帮你扫描石墨空间、判断权限、补导失败文件或保留石墨端复杂协作记录。这些工作必须在石墨导出阶段完成。

    迁移前准备清单

    确认石墨权限、导出范围和文件数量

    团队迁移最怕“导到一半才发现没权限”。迁移前需要确认：个人空间和团队空间分别有多少文件、哪些文件禁止导出、哪些文件属于敏感资料、谁负责补权限、谁负责最终验收。

    团队空间迁移的负责人和验收人

    迁移不是 IT 一个人的工作。推荐每个部门指定一个资料负责人，负责确认目录范围、处理敏感资料、抽查导入结果。IT 或管理员负责工具、权限和批量操作。

    角色 负责内容 关键产出
    迁移管理员 账号权限、导出工具、飞书空间配置 迁移计划和失败文件清单
    部门负责人 确认迁移范围和敏感资料 目录确认表
    内容验收人 抽查导入后的格式和可读性 验收记录
    飞书管理员 重建知识库、文件夹和成员权限 飞书权限配置表

    从石墨导出到本地

    普通文档导出 Word / Markdown

    迁移到飞书时，普通石墨文档优先考虑 Word，因为飞书对 doc/docx 导入支持明确，图片也会包含在文件中。如果你的资料主要是知识库文本，也可以导出 Markdown，但要特别检查图片路径。

    表格导出 Excel

    石墨表格迁移到飞书表格时，应导出为 Excel。不要把业务表格导成 PDF 或图片，否则后续无法编辑。导出后先用 Excel 或 WPS 打开，确认公式、筛选、合并单元格和编码没有明显问题。

    按文件夹保留目录结构

    飞书导入后还要重建空间和知识库结构，所以本地导出目录必须清楚。建议按“部门/项目/文档类型”保留文件夹，不要把所有文件平铺到一个下载目录。

    导入飞书云文档

    导入本地文件夹

    飞书支持上传或导入本地文件和文件夹。实际操作时，建议先在飞书中创建目标空间或文件夹，再按部门、项目分批导入。每批导入后立刻抽查，不要等全部完成后才发现格式问题。

    Word、Excel、PDF 的处理方式

    本地格式 飞书处理建议 迁移后检查
    Word doc/docx 导入为飞书在线文档 标题层级、图片、列表、链接
    Markdown md 导入为在线文档或作为知识库文本 图片路径、代码块、表格
    Excel xlsx 导入为飞书表格或多维表格 公式、日期、筛选、权限
    PDF 作为定稿附件上传 可预览、可搜索、权限正确
    PPTX 导入或上传为演示资料 版式、字体、图片位置

    开放平台 API 适合什么团队

    如果团队有开发能力，可以研究飞书开放平台的导入任务 API。官方接口是异步任务：先上传文件获取 token，再创建导入任务，最后查询导入结果。它适合做自动化批量迁移，但需要应用权限、文件夹 token、限频控制和失败重试逻辑。

    API 不是无门槛方案

    飞书开放平台导入任务对文件扩展名、文件 token、目标云文档类型、挂载点权限和频率都有要求。普通团队优先使用网页导入；开发团队再考虑 API 自动化。

    迁移后格式验收

    导入飞书后，不要只看页面数量。要按文档类型抽查，尤其是图片、表格、内部链接和权限。

    标题、图片、表格、链接检查

    1. 抽查 10 到 20 篇普通文档，确认标题层级和列表样式。
    2. 抽查所有关键表格，确认公式结果、日期格式和筛选视图。
    3. 搜索旧的 shimo.im 链接，决定是否替换为飞书页面链接。
    4. 检查图片是否内嵌成功，尤其是教程、方案和周报。
    5. 确认每个部门能看到自己的文件夹，不能看到无关敏感资料。

    评论、历史版本、@提及的不可迁移项

    石墨中的评论、历史版本、协作记录和@提及通常不会作为正文迁移到飞书。对于审计、客户交付或争议处理相关资料，迁移前应截图或导出关键记录。

    团队迁移避坑

    分批迁移与回滚窗口

    建议先迁移一个低风险项目目录作为试点，再迁移一个部门，最后全量迁移。每一批都要记录开始时间、结束时间、导出数量、导入数量、失败文件和验收人。石墨端资料至少保留 30 天，作为回滚窗口。

    迁移完成后的权限重建

    飞书和石墨的权限模型不同，不能假设权限会自然继承。迁移完成后，按飞书空间、知识库或文件夹重新配置成员权限。敏感资料建议先小范围开放，再由负责人确认后扩大访问范围。

    常见问题

    飞书现在能直接导入石墨文档吗？

    飞书有官方石墨导入说明，但前提通常是先把石墨在线文档下载到本地，再选择对应文件类型导入飞书云文档。不要把它理解成无需导出的账号级一键迁移。

    石墨文档迁移飞书用 Word 还是 Markdown？

    普通团队迁移优先 Word，因为图片和排版更容易保留；知识库文本可以用 Markdown；表格必须单独导出为 Excel，再导入飞书表格或多维表格。

    石墨文件夹导入飞书后目录会保留吗？

    如果本地导出时已经保留文件夹结构，并按文件夹上传或分批导入飞书，目录更容易保留。建议先用一个小目录测试，再迁移全量资料。

    石墨表格迁移到飞书会丢公式吗？

    基础公式通常更容易迁移，复杂公式、跨表引用、条件格式和数据验证规则需要重点检查。关键业务表建议由表格负责人逐个验收。

    评论和版本历史能迁移到飞书吗？

    通常不能完整迁移。评论、历史版本、协作记录和@提及属于石墨平台协作信息。重要内容应在迁移前截图、导出或整理成单独说明文档。
